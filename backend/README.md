# Karma Community Backend ⚙️

Серверна частина Telegram Mini App для гейміфікації спільноти. Побудована на базі сучасного стеку Node.js з інтеграцією бота та REST API.

## 🚀 Технологічний стек

- **[grammY](https://grammy.dev/):** Високопродуктивний фреймворк для Telegram ботів.
- **[Express](https://expressjs.com/):** Легкий веб-сервер для API ендпоїнтів.
- **[SQLite](https://www.sqlite.org/):** Надійна та швидка вбудована база даних для зберігання карми та повідомлень.
- **[Node.js](https://nodejs.org/):** Середовище виконання (v20+ рекомендовано).

## 🛠️ Функціонал

1.  **Гейміфікація через реакції:** Бот слухає подію `message_reaction` і нараховує карму автору повідомлення, якщо на нього поставили 🔥, ❤️, 👍, 👏, 🏆, 💯 або ⚡.
2.  **Авто-реєстрація:** Користувачі автоматично додаються до бази при старті бота або при першому запиті профілю через API.
3.  **REST API:**
    - `GET /api/leaderboard`: Повертає ТОП-50 користувачів за рівнем карми.
    - `GET /api/user/:id`: Повертає детальний профіль користувача та його точне місце в рейтингу.
4.  **Справедливий рейтинг:** Алгоритм враховує не лише кількість карми, а й дату реєстрації (ID) для усунення дублікатів у ранжуванні.

## 📦 Встановлення та запуск

### 1. Підготовка
Переконайтеся, що у вас встановлено Node.js та npm.

### 2. Клонування та залежності
```bash
cd backend
npm install
```

### 3. Конфігурація (.env)
Створіть файл `.env` у папці `backend/` та додайте наступні змінні:
```env
BOT_TOKEN=ваш_токен_бота
PORT=3015
```

### 4. Запуск
```bash
# Локальний запуск
node server.js

# Використовуючи системний сервіс (рекомендовано для продакшну)
systemctl start karma-bot
```

## 🗄️ Структура БД (SQLite)

- **`users`**: Зберігає ID користувача, юзернейм, ім'я та загальну кількість карми.
- **`messages`**: Тимчасово зберігає зв'язок `message_id` -> `user_id` для коректного нарахування карми автору при отриманні реакції.

## 🛡️ Безпека
Бекенд налаштований на роботу за проксі (Vite Proxy) через основний домен `winner.srvrs.top`, що забезпечує захист від CORS проблем та приховує внутрішню структуру API.
